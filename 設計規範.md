# 設計規範

> 本文件根據 20 張不同 App（健康追蹤、旅遊規劃、日曆、房產、任務管理等）的設計參考圖歸納共通元素，作為「AI伴侶網頁」專案的設計系統指南。這些參考圖風格各異，但共享一套現代行動 App 的設計語言——本文件即在提煉這套共通語言，而非複製任何單一款式。

## 1. 設計理念

- **卡片式資訊架構**：所有內容都封裝在圓角卡片中，彼此以留白分隔，不使用實線邊框做分割。
- **淺色底、深色字**：背景以米白／淺灰／近白為主，卡片略淺於或等於背景色，靠陰影而非邊框產生層次。
- **數字即主角**：關鍵數據（心率、卡路里、步數、金額）用超大粗體數字呈現，單位與說明文字縮小、降灰。
- **一類別一色彩**：不同資料類型（心率、水分、睡眠、活動）各自綁定一個強調色，跨頁面保持一致，兼具辨識與美觀。
- **圓潤剋制**：大圓角、圓形按鈕／頭像、膠囊狀分段控制器，整體避免尖角與高飽和大面積色塊。

## 2. 色彩系統

### 2.1 基礎色

| 用途 | 色值建議 | 說明 |
|---|---|---|
| 頁面背景 | `#F5F4F0` ～ `#FAFAFA` | 米白／淺灰，非純白，降低刺眼感 |
| 卡片背景 | `#FFFFFF` 或比背景深一階的淺灰 | 搭配陰影 `0 2px 12px rgba(0,0,0,0.05)` |
| 主要文字 | `#1A1A1A` ～ `#111111` | 接近黑但不用純黑 |
| 次要文字／標籤 | `#8A8A8A` ～ `#9B9B9B` | 用於單位、說明、日期等輔助資訊 |
| 分隔線（少用） | `#EDEDED` | 僅在列表項目間必要時使用，優先用留白取代 |

### 2.2 強調色（依資料類別區分，可依專案調整色相）

| 類別範例 | 建議色 |
|---|---|
| 警示／心率／熱量 | 珊瑚紅、橘紅 `#F97362` |
| 水分／資訊類 | 天藍 `#5AC8E8` |
| 活動／能量 | 黃橘 `#F5A623` |
| 睡眠／專注 | 深紫 `#6C4FCE` |
| 成功／完成狀態 | 綠 `#3DBE6C` |
| 中性強調（CTA按鈕） | 純黑或深墨綠、酒紅等單一主色 |

原則：整體介面維持低飽和中性底色，**每個畫面最多 1～2 個高飽和強調色**作為視覺重點，避免多色混雜。

## 3. 版面與間距

- **圓角半徑**：卡片 16–24px；按鈕／輸入框 12–20px（或全圓角膠囊）；頭像、圖示按鈕皆為正圓形。
- **陰影**：以柔和擴散陰影取代邊框，`blur 12–24px`、透明度 5–8%。
- **卡片內距**：上下左右一致，建議 16–20px。
- **卡片間距**：12–16px，讓卡片有呼吸感。
- **格線**：常見雙欄小卡片＋單欄大卡片混排（如 2 個小數據卡並排 + 下方 1 張長條卡）。

## 4. 字體排印

| 層級 | 用途 | 建議樣式 |
|---|---|---|
| 巨數字 | 核心數據（如 `325.05`、`125 bpm`） | 32–48px，Bold／Semibold |
| 頁面標題 | 如「Body Statistics」「Activities」 | 20–24px，Semibold |
| 卡片標題 | 如「Heart Rate」「Sleep」 | 14–16px，Medium，可搭配小圖示 |
| 內文／說明 | 描述文字 | 13–14px，Regular，灰階 |
| 標籤／單位 | 如 `kcal`、`bpm`、日期縮寫 | 11–12px，Regular，淺灰 |

字體選用無襯線字體（如 Inter、SF Pro、或中文搭配思源黑體／PingFang），數字建議使用等寬或表格數字（tabular figures）以利對齊。

## 5. 元件規範

- **頂部列**：左側圓形大頭貼＋問候語（如「Hi, Alice」），右側 1–2 個圓形圖示按鈕（通知鈴鐺、客服、設定），背景白色圓底浮於淺灰背景上。
- **分段控制器（Segmented Control）**：膠囊外框，選中項目為黑色或深色實心底＋白字，未選中透明底＋灰字。常見於「Week / Month」「Day / Week / Month」切換。
- **搜尋列**：全圓角膠囊，左側放大鏡圖示，右側可加一個獨立的篩選按鈕（方形圓角、強調色底）。
- **底部導覽列**：4–5 個圖示，白色圓角膠囊或懸浮條，當前頁圖示以實心／深色圓底標示，其餘為線性圖示＋灰階。
- **狀態徽章 / Pill**：小型圓角矩形或全圓角，淺色底＋深色文字（如 `LIMITED SLOTS`、`BETA`、`Completed`），字級 10–11px，全大寫或加粗。
- **日期選擇器**：圓形日期格，選中日＝實心深底白字，今日＝外框標示或小氣泡提示（如「今日」「오늘」）。
- **主要按鈕（CTA）**：全圓角膠囊或大圓角矩形，單一主色實心底，白字置中，如「Get Started」「Continue」「Start my trip」。
- **次要按鈕**：白底＋淺灰邊框或無邊框，深色文字，通常與主按鈕並排（如 Cancel / Continue）。

## 6. 資料視覺化

- **圓形／半圓進度環**：用於單一比例型數據（卡路里攝取進度、達成率百分比），環寬適中、留白內圈放核心數字。
- **長條圖**：用於一週／多日對比（步數、活動時間），長條圓角化、依數值大小漸層或雙色區分。
- **折線圖 / 面積圖**：用於趨勢型數據（體重、心率區間），線條平滑、下方可加淺色漸層填色。
- **心跳波形圖（心率專用）**：直條高低起伏模擬心電圖，強調色（紅／珊瑚色）呈現。
- **多環同心圖**：多類別資料疊加在同一圓心（如水分、活動、心率、睡眠合併呈現週摘要）。

## 7. 圖示系統

- 線性圖示（outline）為主，粗細一致（約 1.5–2px stroke），搭配少量實心圖示做「已選中」狀態區分。
- 圖示尺寸統一（20–24px），置於圓形色底容器中作為卡片小標時，容器約 32–40px。
- 表情式／插畫風圖示（如水滴、月亮、火焰）可用於強化資料類別辨識度，但需維持統一插畫風格，不與線性圖示混用在同一層級。

## 8. 落地建議（AI伴侶網頁）

1. 建議先定義 3–5 個「資料／功能類別」（例如：對話、心情記錄、任務、提醒、成就），各自綁定一個強調色，比照上方色彩系統延伸。
2. 網頁版可將行動端的「底部導覽列」轉換為左側或頂部導覽，但保留「當前頁高亮＋圓角容器」的視覺邏輯。
3. 首頁建議採卡片式儀表板佈局：頂部問候語＋頭像，中段 2 欄小卡＋下方長條圖／趨勢卡，符合本規範的資訊層級。
4. 建議建立 Design Tokens（色彩、圓角、陰影、字級）於 CSS 變數或設計系統檔案中，方便日後元件庫統一調用。

## 9. 已落地的具體規則（實際套用紀錄，之後新增功能請延續）

> 本節記錄專案中已經確定套用、非抽象通則的具體做法，來自實際參考圖與調整結果。色彩維持專案既有主題（紫／粉／棕／綠／深色），不套用本文件第 2 節的中性淺色配色。

### 9.1 卡片元件（已套用）

- 卡片圓角統一 16–20px；卡片內距上下左右一致（16px）；陰影統一為柔和擴散陰影，透明度約 5–8%、blur 12–20px，不使用生硬深色陰影（節慶類彈窗如紅包票券除外，可保留較強陰影營造「浮起」效果）。
- 「巨數字」類統計數字分兩級，同一級距內必須是完全相同的數值，不是落在範圍內就好（曾經發生同級距內 22px / 24px / 30px 混用，肉眼看得出不一致）：
  - **Tier A・首頁單一主要倒數（Hero）**：32px Bold。目前只有 `.cnt-days`（首頁「距離◯天」）屬於這一級。
  - **Tier B・清單或多欄並列的統計數字**：24px Bold。包含 `.btc-stat-num`（行事曆/打卡統計列）、`.qa-stat-num`（每日問題統計）、`.cnt-milestone-days`、`.cnt-anni-days`（紀念日倒數清單）等所有「一次列出多筆、每筆各自一個數字」的情境。
  - 之後新增任何「大數字」顯示，先判斷是 Hero（單一、全頁最主要）還是清單類（並列多筆），對號套用對應 Tier 的 px 值，不要自訂新的中間值。

### 9.2 月曆格子（行事曆／每日打卡共用 `.btc-grid` / `.btc-day`）

參考：日期範圍選擇器 App（如 Airbnb 訂房日曆）＋健康追蹤 App 的圓形日期格。

- 每一天預設就是一個 36px 實心圓（淺色底），數字 16px Bold，不是等到「今天」或「已打卡」才出現圓圈。
- 狀態優先順序（後者覆蓋前者）：一般（淺色圓）→ 今天（外框圓圈，色彩=主題色）→ 已打卡/事件（實心色圓＋白字）→ 使用者點選查看中（實心深色圓＋白字，取代原本整格淡色底）。
- 篩選標籤（如「事項／紀念日／生理期」）改用「彩色圓點 + 文字」的膠囊標籤，圓點顏色對應各分類既有的強調色變數（`--event-color`、`--birthday-color`、`--period-color` 等），不用 emoji 當前綴。

### 9.3 日期範圍視覺（生理期等跨日資料）

參考：Airbnb 訂房日曆的「選中區間」樣式——只有起訖日是實心圓，中間的日期只是坐在淡色色帶上的一般文字，不是每天都各自畫一個實心圓。

- 色帶（`::before`）高度、位置需對齊當下日期圓圈的實際大小與位置（目前為 36px 圓圈時，色帶 `top:0; height:36px`）。
- 色帶左右延伸值必須對齊 `.btc-grid` 的實際欄距（gap）：延伸值只能是 gap 的一半，兩個相鄰色帶才會剛好在縫隙中點接壤，不會重疊出現一條更深色的接縫線。目前欄距 2px，延伸值為 `-1px`；之後若調整欄距，記得同步換算這個值。
- 若跨日範圍橫跨多週（換行），每一行的行首/行尾會各自形成一次「起訖日實心圓」視覺斷點，這是行事曆格線本身的限制、非資料錯誤，維持這個效果即可，不需要特別處理成單一色帶橫跨整個月曆。
- 多天事項（有 `endDate` 的行事曆事項）套用同一套「起訖實心圓＋中間淡色帶」樣式，色帶用該事項自己的分類色／自訂色（`--ev-bar-c`）。因為同一天有可能同時符合生理期與多天事項兩種範圍，多天事項的色帶改用 `::after`（生理期維持 `::before`），避免兩個偽元素互相覆蓋。

### 9.4 健康紀錄的圖示化呈現（每日打卡「查看某天紀錄」彈窗）

參考：Hydrate 類水分追蹤 App 的圓環量表、心情追蹤 App 的表情符號列。

- 有明確「數值／目標」概念的項目（睡眠時數、飲水量）用圓環量表（`conic-gradient` 圓環＋內圈顯示數字），不是純文字。量表百分比＝目前值 / 參考基準（睡眠以 8 小時、飲水以 2000mL 為基準，超過封頂 100%）。
- 圓環顏色依資料類別區分：睡眠沿用主題色，飲水固定用天藍 `#5AC8E8`（對應第 2.2 節「水分／資訊類」建議色，不受主題色影響）。
- 有「多筆同類型紀錄」的項目（心情）用小圓形彩色表情符號徽章呈現，顏色沿用既有的分級色彩對照表，不用文字色塊。
- 有「狀態別」而非數值的項目（排便正常/異常/沒有）用彩色圓角徽章（綠色打勾／紅色驚嘆號／灰色橫線），不用純文字。
- 飲食、飲料、生理期文字細節、備註等維持原本的圖示＋標籤／文字呈現，暫不強制套用量表化。

### 9.4a 月曆日期圓圈的心情配色（每日打卡 `.btc-day-num`）

參考：Dailybean、MOODA 等心情追蹤 App——月曆上每一天直接用情緒色塊呈現，不是額外加一個小圓點。

- 當天若有心情紀錄，日期圓圈的底色直接改成情緒色（用既有的 `MOOD_COLORS` 對照表），取代原本「已打卡＝主題色」的填色邏輯。
- 當天若有「多筆不同心情」（一天內心情有變化），改成 `linear-gradient(135deg, ...)` 依紀錄時間順序排列多個顏色，不要用平均值或只取頭尾兩色。
- 這個顏色用行內樣式（inline style）直接蓋在 `.btc-day-num` 上，不透過 CSS class，因為顏色是逐日動態計算的，用 class 沒辦法涵蓋所有組合。
- 因為主圓圈已經承載心情色，原本圓圈下方那顆小小的「心情圓點」（`.btc-day-health` 裡的第三顆點）予以移除，避免重複；睡眠、飲水的小圓點維持不變。
- 修改任何「記錄健康資料」的輸入事件時，記得同時呼叫 `renderCiCalGrid()` 讓月曆即時反映，不要只更新輸入表單本身（心情按鈕原本漏了這個呼叫，數值已存進去但月曆要等重新整理才會顯示）。

### 9.5 週總覽小工具（彩色膠囊長條圖）

參考：多款健康／習慣追蹤 App 常見的「本週多指標一覽」卡片——每個指標各自一根實心圓角長條，長條內底部顯示百分比，顏色依指標各自不同，並排比較。

- 用於「一次比較多個不同類別指標」的場景（例如打卡率、睡眠達成率、飲水達成率、心情指數），跟第 9.4 節「單一指標的圓環量表」是不同情境：圓環量表用在查看單一天的單一數值，膠囊長條圖用在一次總覽多個指標的相對高低。
- 長條軌道（`.ci-week-bar-track`）固定寬度、圓角膠囊造型；填充長條（`.ci-week-bar-fill`）依百分比決定高度，最低給 12% 高度避免 0% 時看起來像消失。
- 已建立的四個指標與對應色彩，之後如需擴充新指標請延續同一套配色邏輯（不套用主題色的指標固定用專屬色，套用主題色的指標則跟隨 `--companion`）：
  - 打卡率 → `var(--companion)`
  - 睡眠達成率 → `#6C4FCE`（對應第 2.2 節「睡眠／專注」建議色）
  - 飲水達成率 → `#5AC8E8`（同 9.4 節飲水圓環色，保持跨元件一致）
  - 心情指數 → `#F5A623`（對應第 2.2 節「活動／能量」建議色）
- 這類「多指標橫向比較」的長條圖统一用膠囊造型＋百分比標示，不用一般直角長條圖，也不混用折線圖。
